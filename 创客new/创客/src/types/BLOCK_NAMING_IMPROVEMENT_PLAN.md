# 积木命名优化方案

## 优化原则
1. **直观性**：名称应该一看就懂，避免专业术语
2. **完整性**：清楚表达积木的主要功能
3. **一致性**：同类积木使用相似的命名模式
4. **简洁性**：在清晰的前提下尽量简短

### 命名改进原则总结

1. **动作明确化**
   - "保存状态" → "保存状态到本地"
   - "粒子特效" → "发射粒子特效"

2. **避免专业术语**
   - "瞬移" → "移动"
   - "缓动" → "平滑移动"
   - "光线检测" → "射线检测"

3. **操作清晰化**
   - "指南针" → "获取方向"
   - "加速度计" → "获取加速度"
   - "录音" → "录制音频"

4. **描述通俗化**
   - "并行执行" → "同时执行"
   - "等待条件" → "等待直到"
   - "相对角度" → "计算角度"

## 实施建议

### 优先级
- 🔴 **高优先级**（必须修改）：专业术语过多，用户难以理解
  - 缓动移动 → 平滑移动到
  - 并行执行 → 同时执行
  - 相对角度 → 计算角度

- 🟡 **中优先级**（建议修改）：命名不够完整或清晰
  - 保存状态 → 保存状态到本地
  - 瞬移到坐标 → 移动到坐标
  - 切换造型 → 切换外观

- 🟢 **低优先级**（可选修改）：命名已经可以接受，但可以更好
  - 震动 → 设备震动
  - 录音 → 录制音频
